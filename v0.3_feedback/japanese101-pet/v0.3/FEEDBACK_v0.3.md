## §A. Bug Fixes(PR1)

### A1. 🎧 きいて クイズ 沒有選項(critical)
- 症狀:玩 きいて クイズ 看不到任何選項可點
- 可能原因:渲染邏輯沒讀到 options 陣列、或 options 為空時的 fallback 沒寫
- 修法:檢查資料載入流程,確認 options 至少 4 個被傳到 UI;若 options 為空必須拋錯而非靜默渲染空白

### A2. ✍️ かな かいてみよう 完成後「オレンジ = おてほん」空白
- 症狀:寫字完按確認 → 出現星星與「つぎへ」,中間有「オレンジ = おてほん」這塊是空白
- 推測:應該是要顯示**範本字 vs 孩子寫的字**對照,但範本沒渲染
- 修法:該區塊應顯示「✏️ おてほん(範本)| 🟠 きみのじ(你的字)」並排對照,讓孩子看自己寫的跟範本的差別

### A3. ✏️ だくてん ドラッグ 答對後出現兩個濁點
- 症狀:と → ど 拖完濁點,結果 ど 上面有兩個濁點(゛゛)
- 原因:UI 渲染了「原字 ど + 拖過去的濁點」,但濁點已經內建在 ど 字裡了
- 修法:答對後,**只顯示 target 字本身**(ど),不要再額外渲染拖入的濁點 sprite

### A4. ✍️ みほん 動畫不照筆順(critical 學習效果問題)
- 症狀:現在的範本動畫是「整個字由上往下淡入」,不是按筆順畫
- 學習問題:孩子看不到正確筆順,寫字學習效果幾乎為零
- 修法:每個假名需要**正確的筆順資料**,動畫要逐筆畫繪製
- **資料需求**:每個假名要有 strokes 陣列,每個 stroke 是 SVG path
- **參考**:
  - 提供的 PDF `hiragana_color_romaji.pdf`(家長已附)
  - https://www.youtube.com/watch?v=wD3FJgij79c
  - 或用開源資料:KanjiVG(https://kanjivg.tagaini.net/),有所有假名的 SVG 筆順,授權是 CC-BY-SA 3.0
- **建議實作**:用 KanjiVG 的 SVG path 資料(MIT-compatible),解析每個 `<path>` 為一筆,用 `stroke-dasharray` + `stroke-dashoffset` 做逐筆動畫

## §B. UI/UX 體驗改善(PR2)

### B1. 按鈕點擊音效
- **資源來源**:https://taira-komori.net/freesoundtw.html(家長指定)
- **授權**:該網站音效是「フリー素材」,免費可商用,但需確認每個音效個別授權
- **建議下載 3-5 個音效**:
  - `button_click.mp3`(主要點擊)
  - `correct.mp3`(答對輕音)
  - `level_complete.mp3`(關卡完成)
  - `page_turn.mp3`(換頁過場)
  - `bubble_pop.mp3`(泡泡爆開)
- **放置位置**:`public/assets/sounds/`
- **管理**:用 Howler.js(SPEC §3 已指定)做音效池,在 `src/lib/audio.ts` 集中管理
- **音量控制**:家長端可獨立調整音效音量(0-100%),預設 60%
- **無障礙**:加「靜音」開關
- **授權標註**:每個音效在 `public/assets/sounds/LICENSE.md` 列出來源 URL 與授權

### B2. 完成關卡後可重玩,**不阻擋進度**
- 目前狀況推測:クリア ✓ 後可能擋住該關
- 規格:
  - クリア ✓ 的關卡仍可點進去重玩
  - 重玩給少量 XP(原本的 50%,第二次 25%,第三次以上 10%)
  - 已クリア 的關卡在地圖上不阻擋,**下一關永遠可進**
  - 玩家在地圖上自由前進到任何已解鎖關卡

### B3. 點按鈕進入下一關有過場動畫
- 目前狀況:點 → 直接切到下一關,無過場
- 規格:
  - 點「つぎへ」按鈕後,顯示**短過場動畫**(0.8-1.2 秒)
  - 建議元素:夥伴跳一下 + 場景滑動 + 「クリア!」字樣浮現 + 星星撒落
  - 過場期間禁用所有互動(避免連點)
  - 過場結束後才切到下一關 UI

### B4. 🫧 かな キャッチ 三項調整
- **B4.1 泡泡降落速度更慢**
  - 目前 `fallSpeed` 預設 60 → 改為 **40**(新手)/ 55(進階)
  - 在 level config 中可覆寫
- **B4.2 泡泡到底不消失,反彈回上方**
  - 原本:泡泡到底 = miss + 消失
  - 改為:泡泡到底 = **反彈回畫面上方**,繼續落下,直到孩子點到或時間到
  - 規格:反彈動畫 0.3 秒,反彈後重新加速下落
  - 沒有 miss 計算,只有「點到 = 對 / 點到 = 錯」
  - 一個 round 改成「答對 N 題就過」而不是「N 題後結束」
- **B4.3 答案泡泡顏色策略**
  - 新手村(L0 謎の卵):**正確答案的泡泡為黃色**(教學提示)
  - 其他所有關卡:**所有泡泡同色**(避免孩子靠顏色判斷)
  - 由 level config 控制:`highlightCorrectAnswer: true | false`(預設 false)

### B5. 寫字遊戲 emoji 跟字相關
- 目前狀況:emoji 是通用的(例如 ⭐ ✏️)
- 改成:**emoji 跟該字最常見的詞相關**
- 範例對照:
  - き → 👂(きく=聽)
  - さ → 🌸(さくら=櫻花)
  - あ → 🐜(あり=螞蟻)
  - い → 🐶(いぬ=狗)
  - う → 🐴(うま=馬)
- 資料:在 `data/kana.json` 每個 kana entry 加 `emoji_hint` 欄位

## §C. 多玩家 Profile + 進度保存(PR3)

### C1. 規格
- **規模**:2-3 個玩家(家中小孩、可能加家長體驗用)
- **儲存**:純本地 IndexedDB,**不上雲端**(完全離線、保護兒童隱私)
- **認證**:**無密碼**,因為是家裡共用裝置

### C2. 玩家選擇 UI
首頁進入前先到「玩家選擇頁」:
```
   ╔══════════════════════════╗
   ║  だれですか?              ║
   ║                          ║
   ║  [🦊 小明]  [🐱 小花]      ║
   ║                          ║
   ║       [➕ あたらしく]      ║
   ║                          ║
   ║       [⚙ おうちのひと]     ║   ← PIN 入家長端
   ╚══════════════════════════╝
```

### C3. 資料結構
```ts
// IndexedDB store: profiles
interface Profile {
  profile_id: string;          // uuid
  name: string;                // 名字(平假名為主)
  avatar_id: string;           // 從預設 8 個頭像選一(SVG)
  created_at: string;
  last_played: string;
}

// IndexedDB store: profile_state (一個 profile 一筆)
interface ProfileState {
  profile_id: string;          // 與 Profile.profile_id 對應
  pet_state: PetState;
  adventure_progress: AdventureProgress;
  practice_xp: number;         // 自由練習累積 XP
  settings: { mic_mode: ..., font_size: ..., ... };
}
```

### C4. 切換玩家
- 首頁固定有「ほかのプレイヤー」按鈕,可回到玩家選擇頁
- 切換時不刪除其他玩家的進度,只是切換目前活躍的 profile

### C5. 新增玩家流程
- 點「➕ あたらしく」
- 輸入名字(平假名輸入,孩子能用)
- 從 8 個預設頭像選一個(SVG,Claude Code 畫)
- 直接進入新手村,從 L0 開始

### C6. 家長端管理
- 在家長端可以看到所有 profile 的進度
- 可以重置某個 profile、可以刪除某個 profile(有確認對話框)
- 可以匯出/匯入 profile 資料(JSON 下載,供未來換裝置用)

## §H. 自然發音音檔(PR6)

### H1. 問題
家長反映目前 Web Speech API 的日文 TTS 太機械,孩子聽不出語感。

### H2. 三個方案選一

**方案 A:在 Web Speech 中指定自然 voice(免費,程式碼修改)**
- Web Speech 在 Mac/iOS 有 `Kyoko`(女)、`Otoya`(男);Windows 有 `Haruka`、`Sayaka`
- 程式碼指定優先使用:`Kyoko` > `Otoya` > default
- 缺點:Android/Linux 仍可能機械

**方案 B:VOICEVOX(免費自然,需要部署)**
- https://voicevox.hiroshiba.jp/ — 商用免費的自然日語 TTS
- 在 build time 預先生成所有需要的音檔(每個 kana + 每個 word)
- 存到 public/assets/audio/,執行時直接播
- 優點:音檔自然、可離線、不依賴瀏覽器
- 缺點:檔案空間(每個音檔 ~10KB,71 詞 + 71 假名 ≈ 1.5MB)

**方案 C:預錄音檔(家長或朋友錄,最自然但需要時間)**
- 找會日文的朋友錄製,放到 public/assets/audio/
- 優點:最自然
- 缺點:需要 1-2 小時錄音

**家長指引的網站**(https://3web.ne.jp/english/jpncntrl_china/aiueo/)
⚠️ 這個網站的音檔有版權,不能直接抓來用。家長提供這個只是要展示「我要這種自然度的音」,不是說要用他們的檔案。

### H3. 我的建議
**方案 B(VOICEVOX)** — 一次性 build time 工作,執行時穩定離線,音質最好平衡。

PR6 中:
1. 加 `scripts/generate-audio.ts` 用 VOICEVOX API 批次生產所有需要的音檔
2. 結果存 `public/assets/audio/<id>.wav`
3. `audio.src` 從空字串改為實際路徑,程式優先用音檔,沒有才 fallback TTS
